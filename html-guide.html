🏗️ HTML Document Structure

<!DOCTYPE html>
Declares this as an HTML5 document
<html lang="en">
Root element containing entire document
<head>
Metadata: title, charset, stylesheets, scripts (not visible on page)
<body>
Visible content: text, images, videos, forms, everything users see
📄
DOCTYPE
Declares HTML5 document type
🌍
<html>
Root element of the document
🧠
<head>
Metadata and document configuration
💪
<body>
Content visible in the browser
Basic Structure: Every HTML document starts with <!DOCTYPE html>, then has an <html> tag containing <head> and <body> elements.

📦 Common Elements

📦
<div>
Generic container / block element
📝
<p>
Paragraph text
🔗
<a>
Hyperlink
🖼️
<img>
Image element
📊
<h1> - <h6>
Headings (levels 1-6)
✂️
<span>
Inline text container
📋
<ul> <li>
Unordered lists
🔢
<ol> <li>
Ordered lists
Block vs Inline: Block elements take full width (<div>, <p>). Inline elements only take needed space (<span>, <a>).

📋 Forms & Input

📝
<form>
Form container
⌨️
<input>
Text/email/password/number fields
📄
<textarea>
Multi-line text input
🏷️
<label>
Form input labels
🔘
<button>
Clickable button
📌
<select> <option>
Dropdown menu
Form Submission: Use the <form> element with method="POST" or method="GET" and an action attribute to send data to a server.

🔤 Semantic HTML5

🎯
<header>
Page or section header
🧭
<nav>
Navigation links
⭐
<main>
Main content of the page
📰
<article>
Self-contained content
📑
<section>
Thematic content grouping
📌
<aside>
Sidebar or tangential content
🔚
<footer>
Page or section footer
🖼️
<figure> <figcaption>
Images with captions
Semantic HTML: Using semantic elements (<header>, <nav>, <main>, <footer>) improves SEO, accessibility, and code readability.
