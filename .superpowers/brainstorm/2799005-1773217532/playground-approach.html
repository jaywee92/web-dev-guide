Interactive Playground — Which approach?

For every CSS topic, the bottom "Playground" section gets pre-filled editable examples. How should it work?

A

Live Code Editor + Preview

Split layout: editable Monaco/CodeMirror code on the left (pre-filled per topic), live HTML preview on the right that updates as you type. Like a mini CodePen embedded in the topic page.

style.css
.box {
background: #3b82f6;
padding: 24px;
}
Hello World
✅ Most interactive   ⚡ Instant feedback   📦 Needs iframe sandbox
B

Tabbed Editor (CSS + HTML + Preview)

Three tabs: CSS code, HTML code, and Preview. Each topic pre-fills both CSS and HTML. User edits either tab, preview updates live. Mobile-friendly, less overwhelming than split.

C

Enhanced Visual Controls + Code Mirror

Keep the existing slider-based visual controls but add a live CSS code output panel below. Sliders generate readable CSS code you can then copy or edit. Bridge between sliders and raw code.

Controls

border-radius 8px
Generated CSS
.box {
padding: 24px;
border-radius: 8px;
}
🔧 Least new code   🎓 Visual → code bridge   ⚠️ No free-form editing
Context: Every CSS topic (box model, flexbox, typography, etc.) gets a pre-filled example relevant to that topic. The example shows a real-world use case you can tweak.
