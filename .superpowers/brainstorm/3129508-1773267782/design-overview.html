← Hover über eine Kategorie
Render-Schichten (eine Komponente: GalaxyWithRays)
1
Galaxy Stars (Basis)
~400 zufällige Sternpunkte, sanftes Twinkle, immer sichtbar. Farbe: dim white/blue-grey.
2
Illuminated Stars (interaktiv)
Dieselben Sterne, neu gezeichnet in Kategoriefarbe wenn sie im Strahlenbereich liegen. Overlay-Modus.
3
Light Ray Beams
5–9 Strahlenfächer von links/rechts. Farbe = Kategoriefarbe. Seite = Cursor-X. Opacity fade on hover.
Architektur

GalaxyWithRays.tsx (neu)

  • Ersetzt AuroraBackground in Home/index.tsx
  • Nimmt ref-Handle: setHover(color|null, side)
  • Einzel-Canvas, CPU-Stars als Float32Array (einmalig generiert)
  • RAF-Loop: stars → illuminated stars → rays
  • Smooth opacity interpolation (easeOut)

CategoryGrid.tsx (Änderung)

  • Erhält galaxyRef prop von Home
  • onCardHover: ruft galaxyRef.setHover(color, side)
  • Side = cursor X relativ zu viewport center
  • Keine State-Changes = kein Re-render
  • onCardLeave: galaxyRef.setHover(null, null)

Home/index.tsx (Änderung)

  • Erstellt galaxyRef = useRef()
  • Übergibt ref an GalaxyWithRays
  • Übergibt ref an CategoryGrid
  • AuroraBackground entfernt

Canvas Rendering

  • Pure 2D Canvas API (kein WebGL nötig für Sterne + Rays)
  • Stars: Array von {x,y,r,baseAlpha,phase} — einmalig init
  • Ray-Bereich-Check: Winkel + Abstand zur Ray-Achse
  • Illuminated stars: gleiche Position, Farbe = lerp(white→categoryColor)

User hovert

Side ermitteln

cursor.x < viewport.w/2 → 'left', sonst 'right'

galaxyRef.setHover

color=#5b9cf5, side='left'

Canvas RAF

→
