Vorschlag 01 / 04
Scrollytelling — Sticky Live-Demo
Während der Nutzer die Erklärung liest, bleibt ein Live-Demo-Fenster rechts immer sichtbar und aktualisiert sich auf den aktuellen Schritt. Kein Hin- und Herspringen mehr zwischen Text und Animation.
web-dev-guide / topic / css-box-model

① Content Box

Das Innerste des Box-Models. Hier fließt dein Text und deine Bilder. Breite und Höhe beziehen sich standardmäßig nur auf diesen Bereich.

width: 200px;
height: 80px;

② Padding

Innenabstand zwischen Content und Border. Nimmt die Hintergrundfarbe des Elements an und vergrößert die klickbare Fläche.

padding: 16px 24px;

③ Border

Der Rahmen um Padding und Content. Gehört standardmäßig zur Gesamtgröße (box-sizing: content-box).

border: 2px solid currentColor;

④ Margin

Außenabstand — schiebt andere Elemente weg. Hat keine Hintergrundfarbe und ist nicht Teil der Elementgröße.

margin: 24px auto;

🧠 Quick Check Frage 1 / 3
Was gibt die folgende Funktion aus?
function counter() {
  let count = 0
  return () => ++count
}
const inc = counter()
console.log(inc(), inc())
Korrekt! 🎉 — Die innere Funktion schließt über count — auch nach dem Aufruf von counter() bleibt count im Speicher und wird bei jedem inc()-Aufruf inkrementiert. Das ist das Wesen einer Closure.
Nicht ganz. — Richtig wäre B: 1, 2. Die Closure speichert eine Referenz auf count in ihrem lexical environment. Diese Variable lebt solange inc existiert.
