🏭 App Factory Pattern

Creates the Flask app dynamically with different configurations. Enables testing and multi-instance deployment.

app/__init__.py

Python 
from flask import Flask
from flask_sqlalchemy import SQLAlchemy

db = SQLAlchemy()

def create_app(config_name='default'):
    app = Flask(__name__)
    app.config.from_object(
        config[config_name]
    )

    # Initialize Extensions
    db.init_app(app)

    # Register Blueprints
    from .routes.auth import auth_bp
    from .routes.api import api_bp

    app.register_blueprint(auth_bp)
    app.register_blueprint(api_bp)

    return app

run.py

Python 
import os
from app import create_app, db

config_name = os.getenv(
    'FLASK_ENV',
    'default'
)
app = create_app(config_name)

with app.app_context():
    db.create_all()

if __name__ == '__main__':
    app.run(
        debug=True,
        host='0.0.0.0'
    )
✨ Benefits of App Factory Pattern
  • Testable — Multiple app instances for tests
  • Configurable — Different settings per environment
  • Extensible — Blueprints easily added
  • Maintainable — Clear separation of concerns

📋 Blueprints — Modular Routes

Blueprints let you organize routes into separate modules — ideal for large applications.

app/routes/auth.py

Python 
from flask import Blueprint

auth_bp = Blueprint(
    'auth',
    __name__,
    url_prefix='/auth'
)

@auth_bp.route('/login', methods=['POST'])
def login():
    return {'msg': 'Login successful'}

@auth_bp.route('/logout')
def logout():
    return {'msg': 'Logout successful'}

app/routes/api.py

Python 
from flask import Blueprint, jsonify
from app.models import User

api_bp = Blueprint(
    'api',
    __name__,
    url_prefix='/api'
)

@api_bp.route('/users')
def get_users():
    users = User.query.all()
    return jsonify([
        u.to_dict() for u in users
    ])
🎯 Blueprint Routing
  • auth_bp → /auth/login and /auth/logout
  • api_bp → /api/users and other API routes
  • Blueprints are registered in create_app()

⚙️ Configuration

Different settings for Development, Production, and Testing — without code changes.

config.py — Configuration Classes

Python 
import os

class Config:
    """Base configuration"""
    SECRET_KEY = os.getenv('SECRET_KEY', 'dev-key')
    SQLALCHEMY_TRACK_MODIFICATIONS = False

class DevelopmentConfig(Config):
    """Development settings"""
    DEBUG = True
    TESTING = False
    SQLALCHEMY_DATABASE_URI = 'sqlite:///dev.db'

class ProductionConfig(Config):
    """Production settings"""
    DEBUG = False
    SQLALCHEMY_DATABASE_URI = os.getenv('DATABASE_URL')

class TestingConfig(Config):
    """Testing settings"""
    TESTING = True
    SQLALCHEMY_DATABASE_URI = 'sqlite:///:memory:'

config = {
    'development': DevelopmentConfig,
    'production': ProductionConfig,
    'testing': TestingConfig,
    'default': DevelopmentConfig
}

🔒 Virtual Environment (venv)

Isolates Python packages per project — prevents conflicts between projects.

❌ Without venv (Dangerous)
System Python
site-packages/
flask==2.0
requests==2.25
⚠️ All projects share packages — version conflicts!
✅ With venv (Safe)
venv/lib/python3.9/
flask==2.3
requests==2.28
✓ Isolated, project-specific packages

venv Setup & Workflow

Bash – Terminal Commands 
# 1. Create virtual environment
python -m venv venv

# 2. Activate venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate        # Windows

# 3. Install dependencies
pip install flask flask-sqlalchemy python-dotenv

# 4. Freeze dependencies
pip freeze > requirements.txt

# 5. Later: Restore dependencies
pip install -r requirements.txt

# 6. Deactivate venv
deactivate
📦 requirements.txt Beispiel 
Flask==2.3.2
Flask-SQLAlchemy==3.0.3
python-dotenv==1.0.0
gunicorn==20.1.0

✅ Summary — Best Practices

🏗️ Structure
  • Use App Factory for flexibility
  • Blueprints for modular routes
  • Separation: app/, config/, templates/, static/
🔒 Security
  • .env for secrets (not in Git!)
  • venv for dependencies
  • .gitignore for sensitive files
⚙️ Configuration
  • config.py with Development/Production/Testing
  • Environment variables via .env
  • app.config.from_object() to load
📦 Dependencies
  • requirements.txt with pip freeze
  • Lock version numbers
  • pip install -r requirements.txt to restore
🧪 Testing
  • create_app() with test config
  • In-memory SQLite for tests
  • Isolated test environment
🚀 Deployment
  • ProductionConfig with real DB
  • DEBUG=False in Production
  • gunicorn as WSGI server
